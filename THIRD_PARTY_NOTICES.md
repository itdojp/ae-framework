# Third-Party Notices

This file records third-party, upstream, and vendored notices that must remain separate from the repository-standard license.

As of 2026-03-14, the authoritative inventory source is the license-scope audit output:

- `artifacts/reference/legal/license-scope-audit.json`
- `artifacts/reference/legal/license-scope-audit.md`

## Operating rules

- Third-party, upstream, and vendored assets keep their original terms.
- Repository-standard licensing does not override subtree-local or file-local notices.
- Nested `LICENSE*`, `NOTICE*`, and `COPYING*` files discovered by the audit must be reviewed before any relicensing decision.

## Current status

- Repository-standard license: Apache-2.0
- Third-party notice candidate audit: `artifacts/reference/legal/third-party-notice-candidate-audit.json`
- Current candidate status: `no-candidates`
- Nested notice inventory: generated by `pnpm run license:audit:scope`

## Update procedure

1. Run:

```bash
pnpm run license:audit:scope -- \
  --output-json artifacts/reference/legal/license-scope-audit.json \
  --output-md artifacts/reference/legal/license-scope-audit.md
```

2. Review `nestedNoticeFiles` and conditional directories.
3. Add concrete third-party notices here only after provenance is confirmed.
