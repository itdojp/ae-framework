# ExecPlan: Common / 05-pr

## Goal
- Prepare PR summary and release notes for review.

## Inputs
- Implementation diff
- Verification results
- Spec kit / blueprint artifacts

## Outputs
- PR description (Background/Changes/Log/Tests/Impact/Rollback/Related Issues; JP: 背景/変更/ログ/テスト/影響/ロールバック/関連Issue)
- Release notes (if needed)

## Steps
1) Summarize intent and spec links.
2) Enumerate verification results and failures.
3) State impact/rollback and open risks.
4) Update `docs/flows/<flow>.md` trace section.

## Notes
- Keep the PR body format consistent with repo policy.
