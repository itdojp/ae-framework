# Spec Kit Constitution

## Core Principles

### I. Reviewable assurance artifacts
Every feature must preserve requirement, plan, task, contract, and verification references in reviewable artifacts.

### II. Optional integration boundary
Spec Kit artifacts may be consumed by ae-framework, but the repository must not vendor or fork upstream Spec Kit.

## Governance
Bridge findings are report-only until a separate policy change promotes a finding class to enforcement.
