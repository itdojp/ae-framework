# PR Summary — Environment Variables

This document defines the environment variables that control PR summary rendering in `render-pr-summary.mjs` and its workflow callers. / 本ドキュメントは、`render-pr-summary.mjs` とその workflow 呼び出し元で PR サマリ描画を制御する環境変数を定義します。

> Language / 言語: English | 日本語

---

## English

### Purpose

Use these variables to control the output language, verbosity, and adapter alert thresholds for PR summaries generated by `scripts/summary/render-pr-summary.mjs`.

### Variables

| Variable | Values / default | Current behavior |
| --- | --- | --- |
| `ADAPTER_WARN_MAX` | integer, default `0` | If adapter summaries contain more than this number of `warn` statuses, the renderer adds an alert line. |
| `ADAPTER_ERROR_MAX` | integer, default `0` | If adapter summaries contain more than this number of `error` statuses, the renderer adds an alert line. |
| `SUMMARY_MODE` | `digest` or `detailed`, default `digest` | `detailed` is enabled only when the value is exactly `detailed`; any other value falls back to `digest`. |
| `SUMMARY_LANG` | `en` or `ja`, default `en` | The renderer lowercases the value and uses Japanese output only when the value is `ja`. |

### Workflow integration

- `pr-ci-status-comment.yml` is the canonical workflow that sets `SUMMARY_MODE` and `SUMMARY_LANG` before calling the renderer.
- `verify-lite.yml` also renders a digest summary and sets:
  - `SUMMARY_MODE=digest`
  - `SUMMARY_LANG=ja` only when the PR has the `lang:ja` label; otherwise `en`
- `SUMMARY_MODE` is typically selected from label state, where `pr-summary:detailed` enables `detailed` output.
- The renderer itself reads the variables directly from the environment and does not validate workflow intent beyond the fallback rules above.

### Representative workflow snippet

```yaml
env:
  ADAPTER_WARN_MAX: 0
  ADAPTER_ERROR_MAX: 0
  SUMMARY_MODE: ${{ steps.mode.outputs.mode }}
  SUMMARY_LANG: ${{ steps.lang.outputs.lang }}
```

### Operational cautions

- Keep threshold values numeric. Non-numeric values are treated as the default `0`.
- `SUMMARY_MODE` is not an open-ended enum. Only `detailed` changes behavior; all other values become `digest`.
- `SUMMARY_LANG` is similarly strict. Only `ja` produces Japanese output; `en`, unset, or any other value produces English output.
- These variables control renderer output only. Artifact selection and append-stage Markdown blocks are handled by the calling workflow.

## 日本語

### 概要

PR サマリ生成の挙動を制御する環境変数の一覧です。
- `ADAPTER_WARN_MAX` / `ADAPTER_ERROR_MAX`: アダプターの警告/エラー許容数
- `SUMMARY_MODE`: `digest` | `detailed`（通常はラベルで切替）
- `SUMMARY_LANG`: `en` | `ja`（既定 `en`）

ワークフローの `env:` で設定して使用します。

### 変数一覧

| 変数 | 値 / 既定値 | 現行挙動 |
| --- | --- | --- |
| `ADAPTER_WARN_MAX` | 整数、既定 `0` | adapter summary に `warn` がこの件数を超えて含まれると、renderer が alert 行を追加する |
| `ADAPTER_ERROR_MAX` | 整数、既定 `0` | adapter summary に `error` がこの件数を超えて含まれると、renderer が alert 行を追加する |
| `SUMMARY_MODE` | `digest` または `detailed`、既定 `digest` | 値が正確に `detailed` の場合のみ詳細モードになり、それ以外は `digest` にフォールバックする |
| `SUMMARY_LANG` | `en` または `ja`、既定 `en` | 値を小文字化したうえで `ja` のときだけ日本語出力、それ以外は英語出力になる |

### Workflow 連携

- canonical な呼び出し元は `pr-ci-status-comment.yml` で、`SUMMARY_MODE` と `SUMMARY_LANG` を設定して renderer を実行する
- `verify-lite.yml` でも digest summary を生成し、以下を設定する
  - `SUMMARY_MODE=digest`
  - `SUMMARY_LANG` は PR に `lang:ja` ラベルがある場合のみ `ja`、それ以外は `en`
- `SUMMARY_MODE` は通常 `pr-summary:detailed` ラベル有無から決定される
- renderer 自体は環境変数だけを読むため、workflow 側の意図は上記フォールバック規則以外では検証しない

### Workflow での利用例

Usage (in workflow step)
```yaml
env:
  ADAPTER_WARN_MAX: 0
  ADAPTER_ERROR_MAX: 0
  SUMMARY_MODE: ${{ steps.mode.outputs.mode }}
  SUMMARY_LANG: ${{ steps.lang.outputs.lang }}
```

### 運用上の注意

- threshold 系は数値で指定する。数値でない場合は既定の `0` として扱われる
- `SUMMARY_MODE` は実質 2 値で、`detailed` 以外はすべて `digest` になる
- `SUMMARY_LANG` も同様に厳格で、`ja` 以外は英語出力になる
- これらの変数は renderer の出力だけを制御する。artifact の選択や append-stage Markdown の追加は呼び出し workflow 側の責務である
