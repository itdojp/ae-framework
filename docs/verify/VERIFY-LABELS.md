# Verify Labels: Optional Enforcement Toggles

> 🌍 Language / 言語: English | 日本語

---

## English

This repository supports opt-in enforcement via PR labels. By default, the relevant checks remain report-only.

### `enforce-contracts`

- Fails the PR when runtime contracts execution reports any failure (`parseIn` / `pre` / `post` / `parseOut`).
- Equivalent to setting `CONTRACTS_ENFORCE=1` in the CI environment.

### `enforce-formal`

- Fails the PR when model checking finds any failing TLC module or Alloy spec.
- Reads `artifacts/codex/model-check.json` generated by `scripts/verify/run-model-checks.mjs`.
- When the separate formal workflow is actually running, also enables stricter formal-pipeline checks, including Apalache `ran/ok` enforcement and Formal Summary v1 strict validation during formal aggregation.
- On PRs, that formal workflow typically runs only when the `run-formal` label is present; maintainers can also invoke it via `workflow_dispatch`.

### Notes

- Use these labels only on PRs where you explicitly want stricter gating.
- When the labels are absent, the pipeline remains report-only so feedback stays fast.
- You can remove the labels on subsequent pushes to return the PR to report-only behavior.

## 日本語

このリポジトリでは、PR ラベルで opt-in の enforcement を切り替えられます。既定では、対象の check は report-only のままです。

### `enforce-contracts`

- runtime contracts execution が `parseIn` / `pre` / `post` / `parseOut` のいずれかで failure を返した場合に PR を fail させます。
- CI 環境で `CONTRACTS_ENFORCE=1` を設定するのと同等です。

### `enforce-formal`

- model checking で failing TLC module または Alloy spec が見つかった場合に PR を fail させます。
- `scripts/verify/run-model-checks.mjs` が生成する `artifacts/codex/model-check.json` を参照します。
- 別系統の formal workflow が実行されている場合に限り、formal pipeline 側の stricter check も有効化し、Apalache の `ran/ok` enforcement と formal aggregation 時の Formal Summary v1 strict validation を含みます。
- PR では通常 `run-formal` ラベルが付いているときだけ formal workflow が動作し、maintainer は `workflow_dispatch` でも起動できます。

### 注意事項

- これらのラベルは、明示的に stricter gating を求める PR だけに使ってください。
- ラベルがない場合、pipeline は report-only のまま動作し、feedback を高速に維持します。
- 後続 push でラベルを外せば、その PR は再び report-only の挙動に戻せます。
