# Web API サンプル（ドキュメント用）

> このファイルはリファレンス用の記述例です。CI では実行されません。

## シナリオ案
- 予約を作成できる（200, body に予約ID）
- 既存予約に対する重複作成は idempotent に扱う（200, 同一ID返却）
- 在庫不足の場合 409 を返し、在庫は変化しない
- 入力バリデーションエラーは 400 + エラーコード/フィールド名を返す

## 例 (Gherkin 例示)
```
Scenario: 在庫があるときに予約を作成できる
  Given 商品 "item-1" の在庫が 5 ある
  When   ユーザー "u1" が item-1 を 2 個予約する
  Then   応答は 200 で予約IDを返す
  And    在庫は 3 になる
```
